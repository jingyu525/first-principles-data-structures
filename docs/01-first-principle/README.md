# 01 计算机的本质：存储与处理

## 第一性原理：计算机到底在干什么？

### 两个核心操作

```
计算机 = 存储 + 处理
```

| 操作 | 含义 | 对应 |
|------|------|------|
| **存储** | 把数据放在哪里 | 内存、磁盘、寄存器 |
| **处理** | 对数据做什么 | CPU 计算、IO 操作 |

### 从存储出发推导数据结构

```
问题：如何高效存储数据？

1. 如果数据是连续的 → 数组
2. 如果数据需要频繁插入删除 → 链表
3. 如果数据需要后进先出 → 栈
4. 如果数据需要先进先出 → 队列
5. 如果需要快速查找 → HashMap
6. 如果需要快速找最大/最小值 → Heap
7. 如果数据有层级关系 → 树
8. 如果数据是网状关系 → 图
```

### 从处理出发推导算法

```
问题：如何处理这些数据？

1. 遍历 → O(n)
2. 二分查找 → O(log n)（前提是数据有序）
3. 排序 → O(n log n)
4. 哈希查找 → O(1)（平均）
```

## 数据结构是什么？

**数据结构 = 数据的组织方式 + 对该组织方式的操作**

```go
// 数组：连续内存 + 随机访问
arr := [5]int{1, 2, 3, 4, 5}
arr[2] // O(1)

// 链表：非连续内存 + 顺序访问
type Node struct {
    Val  int
    Next *Node
}
// 找第 k 个元素需要 O(k)
```

---

## 算法是什么？

**算法 = 解决问题的步骤序列**

同一个数据结构可以有不同的算法：

```go
// 在数组中查找元素

// 算法1：线性查找 → O(n)
func linearSearch(arr []int, target int) int {
    for i, v := range arr {
        if v == target {
            return i
        }
    }
    return -1
}

// 算法2：二分查找 → O(log n)，前提是数组已排序
func binarySearch(arr []int, target int) int {
    lo, hi := 0, len(arr)-1
    for lo <= hi {
        mid := lo + (hi-lo)/2
        if arr[mid] == target {
            return mid
        } else if arr[mid] < target {
            lo = mid + 1
        } else {
            hi = mid - 1
        }
    }
    return -1
}
```

## 为什么需要复杂数据结构？

```
简单场景 → 基本数据结构就够了
复杂场景 → 需要组合和优化

例：
- 存 100 条数据 → 数组足够
- 存 1 亿条数据 + 需要范围查询 → B+Tree
- 千万级 QPS + 实时排行榜 → SkipList (Redis ZSet)
```

### 工程中的权衡

| 数据结构 | 内存效率 | 查找速度 | 插入速度 | 典型场景 |
|---------|---------|---------|---------|---------|
| 数组 | 高 | O(1) 随机 | O(n) | Kafka 缓冲区 |
| 链表 | 中 | O(n) | O(1) | LRU |
| HashMap | 中 | O(1) 平均 | O(1) 平均 | Go Map、Redis Hash |
| SkipList | 中 | O(log n) | O(log n) | Redis ZSet |
| B+Tree | 高 | O(log n) | O(log n) | MySQL InnoDB |

## 一页纸地图

```
内存
 ↓
数组 ────────────── 连续内存，随机访问 O(1)
 ↓
链表 ────────────── 非连续，插入删除 O(1)
 ↓
栈 ─────────────── LIFO，函数调用、表达式计算
 ↓
队列 ────────────── FIFO，消息队列、缓冲区
 ↓
HashMap ─────────── Key-Value，O(1) 查找
 ↓
Heap ────────────── 优先队列，TopK、定时器
 ↓
树 ──────────────── 层级结构，BST/AVL/红黑树
 ↓
图 ──────────────── 网状结构，依赖分析
```

## 关键认知

### 1. 没有银弹

没有一种数据结构在所有场景下都是最优的。每种数据结构都是特定场景下的权衡。

### 2. 数据结构决定算法

```
数据结构 → 决定了能做什么操作 → 决定了算法复杂度
```

### 3. 工程 = 数据结构 + 系统约束

```
Redis ZSet = SkipList + 内存约束
MySQL InnoDB = B+Tree + 磁盘IO约束
Kafka = Queue + 顺序读写约束
```
